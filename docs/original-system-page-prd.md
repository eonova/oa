# 鸿奕博 OA 当前系统功能 PRD（按页面）

日期：2026-06-04  
来源：线上 OA 页面走查  
范围：当前登录账号可见页面和菜单。未进入提交或审批动作，不改动线上数据。

## 1. 文档目的

这份文档记录现有 OA 系统的页面和功能模块，用来指导后续做简化版系统。

重点不是复刻所有细节，而是先搞清楚：

- 现有系统有哪些页面。
- 每个页面解决什么事。
- 页面上有哪些入口、字段和操作。
- 哪些功能适合简化版优先做。
- 哪些内容还需要业务人员补充确认。

## 2. 主框架页面

入口：登录后进入 `main.do?method=main`

用途：OA 的统一工作台。用户在这里切换空间、查看个人信息、进入各业务模块。

页面包含：

- 顶部公司名称。
- 搜索框：输入关键字搜索系统内容。
- 签到入口。
- 首页设置。
- 页签设置：跟随系统、打开、关闭。
- 协同模板设置：跟随系统、新经典、新精致、旧版。
- 个人设置。
- 语言设置：中文简体、中文繁体、English。
- 产品导航。
- 薪资查看。
- 关于。
- 退出登录。
- 空间切换：个人空间、单位空间。
- 左侧菜单。
- 中间首页栏目。

权限说明：

- 左侧菜单和业务入口会按账号权限展示。
- 本次看到的是采购助理岗位可见范围，不代表管理员或其他岗位的完整范围。

简化版建议：

- 保留顶部搜索、个人设置、退出登录。
- 空间切换可以先不做，统一做一个工作台。
- 页签设置、模板设置、语言切换、产品导航可后置。

  - 待办事项
  - 通讯录
  - 添加快捷
- 待办中心：
  - 全部待办
  - 知会
  - 领导发的
- 公司公告 / 单位最新新闻 / 我的项目 / 我的任务。
- 跟踪事项。
- 代理事项。
- 我的日程：
  - 新建计划
  - 新建任务
  - 新建事件
  - 日历周视图
  - 会议、事件、任务、领导行程分类
- 天气。
- 我的消息：
  - 全部
  - @我的
  - 回复我的
  - 跟踪事项的
  - 领导的
  - 重要的
- 我的收藏。
- 我的模板。
- 我的报表。
- 首页栏目配置：
  - 编辑栏目
  - 移除栏目
  - 添加栏目
  - 编辑组合栏目
  - 移除组合栏目
  - 添加组合栏目

简化版建议：

- 首页先做“待办、已发、已办、公告、消息、快捷入口”。
- 日程、天气、栏目自由配置可后置。

包含页面：

- 协同 BPM 门户
- 新建事项
- 待发事项
- 已发事项
- 待办事项
- 已办事项
- 流程督办监控

## 4.1 协同 BPM 门户

入口：左侧菜单 > 协同工作 > 协同 BPM 门户。

用途：流程工作入口页。

已确认内容：

- 菜单存在。

待确认：

- 该页面具体是流程看板、流程分类入口，还是 BPM 统计页。

## 4.2 新建事项 / 表单中心

入口：左侧菜单 > 协同工作 > 新建事项，或首页快捷入口 > 新建事项。

用途：选择业务表单，发起审批流程或协同事项。

当前可见表单：

- 采购付款申请单
- 材料采购申请单
- 其他付款申请单
- 通用类报销申请单
- 客户报价申请单
- 出差报告
- 退货申请单
- 差旅类报销申请单
- 请假/调休申请单
- 加班申请单
- 人力需求申请单
- 折让申请单
- 补卡申请单
- 代理采购申请单
- 会议通知
- 离职申请单
- 法律文书审查
- 出货申请单
- 出差申请单
- 供应商报价登记单
- 招待类报销申请单
- 新员工试用期考核表-非销售类
- 新员工试用期考核表-销售类
- 工作报告
- 用印申请单
- 特殊类报销申请单
- 其他采购申请单
- 资金需求计划表
- 会议纪要

主要功能：

- 展示可发起表单。
- 点击表单进入填写页。
- 表单提交后进入审批流程。
- 未提交内容应进入待发事项。

待确认：

- 每个表单的字段。
- 每个表单的审批节点。
- 表单是否支持附件、明细行、金额大写、自动计算。
- 是否支持保存草稿、撤回、加签、知会。

简化版建议：

- 先做一个通用“发起事项”能力。
- 第一批表单建议做：采购付款、材料采购、报销、请假、用印。
- 其他表单先作为配置项或后续迭代。

## 4.3 待办事项

入口：左侧菜单 > 协同工作 > 待办事项，或首页快捷入口 > 待办事项。

用途：查看并处理别人流转到当前用户的事项。

筛选分类：

- 全部待办
- 已超期
- 7 天内超期
- 本部门的
- 领导发我的

操作按钮：

- 转发
- 归档
- 批处理
- 自动处理

列表字段：

- 标题
- 发起人
- 上一处理人
- 发起时间
- 接收时间
- 处理期限
- 催办次数
- 处理状态
- 流程日志
- 当前节点

处理逻辑：

- 点击标题进入事项详情。
- 在详情页完成审批、阅读、意见填写等动作。
- 流程日志用于查看节点流转记录。

简化版建议：

- PC 端保留待办列表和详情查看。
- 审批动作同时支持 PC 和小程序。
- 批处理、自动处理可以后置。

## 4.4 待发事项

入口：左侧菜单 > 协同工作 > 待发事项，或首页快捷入口 > 待发事项。

用途：管理已保存但还没正式发起的草稿事项。

操作按钮：

- 发送
- 编辑
- 转发
- 删除

列表字段：

- 标题
- 状态
- 创建时间
- 流程期限

处理逻辑：

- 编辑后可继续保存或发送。
- 发送后进入已发事项，并按流程流转。
- 删除只删除草稿，不影响已发流程。

简化版建议：

- 保留草稿列表、编辑、发送、删除。
- 转发草稿可后置。

## 4.5 已发事项

入口：左侧菜单 > 协同工作 > 已发事项，或首页快捷入口 > 已发事项。

用途：查看当前用户发起过的流程，跟踪审批进度。

页签：

- 已发事项
- 交接给我的事项
- 交接给他人的事项

操作按钮：

- 转发
- 归档
- 撤销流程
- 编辑流程
- 重复发起

搜索：

- 按标题关键字搜索。

列表字段：

- 标题
- 发起时间
- 当前待办人
- 流程期限
- 是否归档
- 跟踪状态
- 流程日志

处理逻辑：

- 发起人可查看流程当前停在哪个节点。
- 未结束流程可能支持撤销、编辑流程。
- 重复发起用于按历史表单快速再发起一次。

简化版建议：

- 保留已发列表、流程状态、撤回、复制发起。
- 交接事项、归档可后置。

## 4.6 已办事项

入口：左侧菜单 > 协同工作 > 已办事项，或首页快捷入口 > 已办事项。

用途：查看当前用户处理过的事项。

页签：

- 全部已办
- 撤销回退记录
- 交接给我的事项
- 交接给他人的事项
- 他人竞争处理事项

操作按钮：

- 转发
- 归档
- 取回
- 知会

页面选项：

- 同一流程只显示一条。

列表字段：

- 标题
- 发起人
- 上一处理人
- 发起时间
- 处理时间
- 当前待办人
- 处理期限
- 是否归档
- 跟踪状态
- 流程日志
- 当前节点

处理逻辑：

- 用于查历史处理记录。
- “取回”一般用于流程还没被下一节点处理前撤回自己的处理意见，具体规则待确认。

简化版建议：

- 保留已办列表、详情、处理意见记录。
- 取回、交接、竞争处理可后置。

简化版建议：

- 第一版可以做“流程状态看板 + 超期列表 + 催办”。

## 11. 商务管理

入口：左侧菜单 > 商务管理。

用途：处理报价、采购、出货、退货、折让等商务流程。

包含页面：

- 供应商报价登记
- 客户报价申请
- 材料采购申请
- 代理采购申请
- 出货申请
- 折让申请
- 退货申请

主要功能：

- 发起对应业务申请。
- 填写业务明细。
- 按流程提交审批。
- 在已发、待办、已办中跟踪和处理。

待确认：

- 每张表单的字段。
- 是否有商品明细、金额合计、币种、税率。
- 是否要关联客户、供应商、项目、订单。
- 是否需要导出单据。

简化版建议：

- 先做材料采购申请、供应商报价登记、客户报价申请。
- 出货、折让、退货可作为后续业务表单。

用途：处理报销和付款类流程。

包含页面：

  - 通用类

## 19. 权限和角色

当前页面能确认的权限规则：

- 菜单按账号权限展示。
- 表单入口按角色或岗位权限展示。
- 流程操作按钮会按状态和权限变化。
- 管理类页面可能需要更高权限才能看到完整功能。

简化版建议角色：

- 管理员：用户、角色、权限、数据、流程、导出都可操作。
- 经办人：可发起、处理、查看自己相关事项，可导出授权数据。
- 审批人：可处理待办，可查看自己已办和相关流程。
- 只读用户：只能查看授权范围内数据。
